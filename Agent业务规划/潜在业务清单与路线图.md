# 潜在业务清单与路线图（Portfolio + Roadmap）

日期：2026-03-15  
目标：形成“可卖的 Agent 产品线”组合，并给出 0–18 个月的推进路线。

---

## 1) 业务组合（建议先 2+1）

### A. 现金流型（优先）

**A1. 客服工单 Agent（AgentCompany）**
- ICP：有稳定工单量的 SaaS/电商/服务型企业（10–200 人团队）
- 核心指标：一次解决率、平均处理时长、质检合格率、知识覆盖率
- MVP：分流 + 草稿 + 引用来源 + 人审放行 + 写回工单 + 自动沉淀 FAQ

**A2. 对账/报销 Agent（AgentCompany）**
- ICP：交易频繁且对账痛苦的电商/渠道/连锁；或报销量大的中型企业
- 核心指标：对账时长、差异定位时间、错账率、欺诈拦截率
- MVP：票据/流水结构化 → 规则校验 → 差异定位报告 → 人审 → ERP 写回

### B. 增长型（中期）

**B1. 内容与增长流水线（AgentSocial）**
- ICP：内容驱动增长的品牌方/MCN/创业团队
- 核心指标：内容产能、发布频次、素材复用率、投放实验迭代速度
- MVP：多平台改写 + 品牌语气校验 + 素材归档 + 复盘报告

### C. 行业型（选一个作为“深水区护城河”）

**C1. 供应链/跨境单证 Agent（AgentTrade）**
- ICP：外贸/跨境电商/分销；制造企业采购与外贸部门
- 核心指标：询盘响应速度、报价准确率、单证差错率、交付延期率
- MVP：询盘结构化 → 报价草稿 → 条款比对 → 单证清单 → 交付跟踪提醒

---

## 2) 共用平台能力（必须沉淀，否则无法规模化）

1. 连接器：工单/CRM/ERP/邮箱/IM/表格/文档/数据库
2. 权限与审批：RBAC、动作白名单、关键动作二次确认
3. 评测与回放：失败用例库、回放复现、抽检与评分
4. 可观测：trace、成本、时延、成功率、工具失败原因
5. 数据治理：脱敏、加密、租户隔离、审计日志

---

## 3) 路线图（0–18 个月）

### 0–3 个月：拿到 1 个可复用 MVP
- 交付 1 条端到端流程（建议：客服工单 Agent）
- 明确审批点与放行标准（质量阈值、拒答规则、引用来源）
- 最小可观测：每次任务的 trace + 工具调用日志 + 成本

### 3–6 个月：打磨成“可卖的产品”
- 增加 2–3 个关键连接器（按 ICP 系统优先）
- 加入评测集与持续回放机制（每周迭代）
- 多租户 + 基础权限模型 + 管理后台

### 6–12 个月：从单点走向部门级
- 把“例外处理”沉淀为规则与策略
- 支持批处理与定时任务（日报、周报、月结材料）
- 引入模板/工作流版本管理与灰度发布

### 12–18 个月：平台化与生态
- 流程可视化/DSL（让交付团队与客户可配置）
- 插件机制（第三方工具接入、行业模板）
- 合作伙伴交付：咨询/实施/培训体系

## 4) 机会池（可作为待办 Backlog）

### AgentCompany（企业运营）
- 会议纪要 → CRM 写回 → 跟进任务自动创建
- 知识库“自动发现空洞”：根据工单与搜索日志生成待补充条目
- 发票/合同/订单三方匹配的差异定位与证据链输出

### AgentSocial（增长）
- 热点雷达：竞品/关键词监控 → 选题 → 草稿 → 审核 → 发布排程
- 素材实验：变量管理（标题/封面/卖点）→ 结果归因 → 复用建议

### AgentTrade（供应链/跨境）
- 询报价自动表格化 + 供应商比价与备选方案
- 交期/缺料预警：从库存、生产计划、物流节点自动汇总
- 单证差错检查：字段一致性、条款一致性、时间线一致性

### AgentGame（游戏）
- 剧情/任务一致性校验（人物设定、时间线、道具约束）
- Bug 工单归因（日志 + 复现步骤 + 影响范围）
